The Swiss Law Collection serves 53,000+ federal and cantonal laws as static JSON files via GitHub Pages. No authentication, no API key, no rate limits. All endpoints return JSON and are accessible from any HTTP client or browser.

Base URL: https://swiss-law-as-source.github.io

Contents

  1. Publications API — laws by year
  2. Statistics API — per-year per-canton breakdowns
  3. Global Data Files — aggregate stats, tags, law index
  4. Raw Law Files — Markdown with YAML frontmatter
  5. Code Examples — curl, Python, JavaScript
  6. Try It — interactive API explorer

1. Publications API

Lists all laws grouped by the year they entered into force. Use the index to discover available years, then fetch individual year files.

GET /api/v1/publications/index.json
Index of all publication years with total counts.

Response

FieldTypeDescription
yearsint[]Array of years with publications (1802–2066)
total_publicationsintTotal number of law entries across all years
earliest_yearintFirst year in the dataset
latest_yearintLast year in the dataset
{
  "years": [1802, 1803, 1807, "...", 2024, 2025, 2066],
  "total_publications": 53692,
  "earliest_year": 1802,
  "latest_year": 2066
}
curl -s https://swiss-law-as-source.github.io/api/v1/publications/index.json | jq '.total_publications'
GET /api/v1/publications/{year}.json
All laws that entered into force in a given year.

Path Parameters

ParameterTypeDescription
yearintYear (e.g. 2024). See the index for available years.

Response Fields

FieldTypeDescription
date_prefixstringThe year as a string ("2024")
countintNumber of publications in this year
publicationsobject[]Array of law entries (see below)

Publication Object

FieldTypeDescription
datestringEntry-into-force date (YYYY-MM-DD)
sr_numberstringSR number (federal) or systematic number (cantonal)
titlestringFull legal title
scopestring"federal" or "cantonal"
languagestringLanguage code: de, fr, or it
cantonstringCanton code (e.g. "BE") for cantonal laws, empty for federal
pathstringRelative path to the Markdown file
url_mainstringDirect URL to the raw Markdown file on GitHub
systematic_categorystring?Topic hierarchy (cantonal only)
global_categorystring?Global classification code (cantonal only)
category_typestring?Law type: Gesetz, Verordnung, etc. (cantonal only)

      "date": "2024-01-01",

      "url_main": "https://raw.githubusercontent.com/swiss-law-as-source/swiss-law-as-source.github.io/main/ch/be/de/107.1.md",
      "systematic_category": "...",
      "global_category": "...",

2. Statistics API

Per-year, per-canton (or federal) breakdowns. Use this to answer questions like "How many laws does Bern have in 2024?" or "What types of law did Fribourg enact in 2023?"

GET /api/v1/stats/index.json
Index of available years and entities (cantons + CH).

Response

FieldTypeDescription
yearsstring[]Available years ("1802" through "2066")
entitiesstring[]Entity codes (see below)
total_filesintTotal number of year×entity JSON files

Available Entities

CH (Federal) AG (Aargau) AR (Appenzell AR) BE (Bern) BL (Basel-Land) BS (Basel-Stadt) FR (Fribourg) GL (Glarus) GR (Graubünden) LU (Luzern) SG (St. Gallen) SO (Solothurn) TG (Thurgau) VS (Valais) ZG (Zug)
GET /api/v1/stats/{year}/{entity}.json
Statistics for a specific year and entity (canton or federal).

Path Parameters

ParameterTypeDescription
yearstringYear (e.g. "2024")
entitystring"CH" for federal, or a canton code (e.g. "BE")

Response Fields

FieldTypeDescription
yearstringThe year
entitystringEntity code
totalintTotal laws for this year/entity
by_typeobjectCounts by law type (Gesetz, Verordnung, etc.) — cantonal only
by_topicobjectCounts by systematic category — cantonal only
by_global_categoryobjectCounts by global classification code — cantonal only
by_languageobjectCounts by language (de, fr, it)
For CH (federal), the fields by_type, by_topic, and by_global_category are empty objects. These breakdowns are only available for cantonal entities.

Example: Federal (CH)

// GET /api/v1/stats/2024/CH.json
{

  "by_global_category": {},
  "by_language": { "de": 443, "fr": 443, "it": 443 }
}

3. Global Data Files

Large aggregate files covering the entire dataset. Useful for dashboards or full exports, but consider the per-year endpoints for targeted queries.

GET /stats.json
Aggregate statistics across all 53,000+ laws: totals, breakdowns by language, canton, SR category, law type, year, and cross-tabulations (canton×year, type×canton, etc.). 
curl -s https://swiss-law-as-source.github.io/stats.json | jq 'keys'
# ["by_canton", "by_category_type", "by_global_category", "by_language",
#  "by_month", "by_sr_category", "by_systematic_category", "by_year",
#  "by_year_scope", "canton_by_year", "cantonal_laws", "category_type_by_canton",
#  "category_type_by_canton_by_year", "category_type_by_year",
#  "federal_laws", "language_by_year", "total_laws"]
GET /tags.json
Per-law metadata index grouped by scope. Contains every law's ID, title, language, version date, and category fields.
This file is approximately 19 MB. For targeted queries, prefer the per-year publications endpoint.
{
  "federal": [
    { "id": "0.101", "title": "...", "language": "de", "version_date": "...", "sr_category": "...", "source": "..." }
  ],
  "cantonal": {
    "AG": [ { "id": "110.000", "title": "...", "language": "de", "version_date": "...", ... } ],
    "BE": [ ... ],
    ...
  },
  "total": 53692
}
GET /laws.json
Flat list of all laws with SR/systematic number, title, file path, category, and scope. Powers the search page. 
[
  { "sr": "0.101", "title": "Konvention zum Schutze der Menschenrechte...", "path": "ch/0/de/0.101.md", "cat": "0 – Völkerrecht", "scope": "federal" },
  { "sr": "110.000", "title": "Verfassung des Kantons Aargau", "path": "ch/ag/de/110.000.md", "cat": "AG", "scope": "cantonal" }
]

4. Raw Law Files (Markdown)

Every law is stored as a Markdown file with YAML frontmatter. You can fetch any law directly via its raw GitHub URL.

Raw Base URL: https://raw.githubusercontent.com/swiss-law-as-source/swiss-law-as-source.github.io/main/

File Path Patterns

ScopePatternExample
Federal ch/{prefix}/{lang}/{sr_number}.md ch/0/de/0.101.md — SR 0.101 in German
ch/220/de/220.md — SR 220 (Code of Obligations)
Cantonal ch/{canton}/{lang}/{number}.md ch/be/de/107.1.md — Bern law 107.1 in German
ch/fr/fr/110.1.md — Fribourg law 110.1 in French
For federal laws, {prefix} is the integer part before the first dot in the SR number. Examples: SR 0.101 → prefix 0, SR 220 → prefix 220, SR 741.01 → prefix 741. Canton codes are lowercase: be, fr, ag, etc.

Frontmatter Fields

FieldScopeDescription
sr_numberFederalSR classification number (e.g. "101")
systematic_numberCantonalCanton-specific number (e.g. "107.1")
titleBothFull legal title
languageBothde, fr, it, or rm
version_dateBothISO date when this version entered into force
sourceBoth"https://fedlex.data.admin.ch" (federal) or "LexWork" (cantonal)
cantonCantonalTwo-letter code, uppercase (e.g. "BE")
category_typeCantonalLaw type: Gesetz, Verordnung, Reglement, etc.
systematic_categoryCantonalTopic hierarchy
abbreviationFederalShort form (e.g. "BV", "OR")

Example: Federal Law

---
sr_number: '101'
title: Bundesverfassung der Schweizerischen Eidgenossenschaft
language: de
version_date: '1999-06-18'
source: https://fedlex.data.admin.ch
abbreviation: BV
---

# Bundesverfassung der Schweizerischen Eidgenossenschaft

## 1. Titel: Allgemeine Bestimmungen
Art. 1 Schweizerische Eidgenossenschaft
...

Example: Cantonal Law

---
canton: BE
systematic_number: '107.1'
title: Gesetz über die Information und die Medienförderung
language: de
version_date: '2024-01-01'
source: LexWork
category_type: Gesetz
systematic_category: '107 Medien'
---

# Gesetz über die Information und die Medienförderung
...

5. Code Examples

Fetch all laws for a canton in a given year

# Get all Bern (BE) laws that entered into force in 2024
curl -s https://swiss-law-as-source.github.io/api/v1/publications/2024.json \
  | jq '[.publications[] | select(.canton == "BE")]'
import requests

BASE = "https://swiss-law-as-source.github.io"
year = 2024
canton = "BE"

data = requests.get(f"{BASE}/api/v1/publications/{year}.json").json()
laws = [p for p in data["publications"] if p["canton"] == canton]
print(f"{len(laws)} laws for {canton} in {year}")

for law in laws[:5]:
    print(f"  {law['sr_number']} — {law['title'][:60]}")
const BASE = "https://swiss-law-as-source.github.io";
const year = 2024, canton = "BE";

const res = await fetch(`${BASE}/api/v1/publications/${year}.json`);
const data = await res.json();
const laws = data.publications.filter(p => p.canton === canton);
console.log(`${laws.length} laws for ${canton} in ${year}`);

Get statistics for all cantons in a year

# Get total laws per canton for 2024
for entity in CH AG AR BE BL BS FR GL GR LU SG SO TG VS ZG; do
  total=$(curl -s "https://swiss-law-as-source.github.io/api/v1/stats/2024/${entity}.json" | jq '.total')
  echo "$entity: $total"
done
import requests

BASE = "https://swiss-law-as-source.github.io"
year = "2024"

index = requests.get(f"{BASE}/api/v1/stats/index.json").json()
for entity in index["entities"]:
    stats = requests.get(f"{BASE}/api/v1/stats/{year}/{entity}.json").json()
    print(f"{entity}: {stats['total']} laws ({stats.get('by_language', {})})")

Download all law texts for a canton

import requests, time, os

BASE = "https://swiss-law-as-source.github.io"
canton = "BE"
out_dir = f"laws_{canton}"
os.makedirs(out_dir, exist_ok=True)

# Get all years
index = requests.get(f"{BASE}/api/v1/publications/index.json").json()

for year in index["years"]:
    data = requests.get(f"{BASE}/api/v1/publications/{year}.json").json()
    laws = [p for p in data["publications"] if p["canton"] == canton]

    for law in laws:
        # Download the raw Markdown
        resp = requests.get(law["url_main"])
        if resp.ok:
            filename = law["sr_number"].replace("/", "_") + ".md"
            with open(f"{out_dir}/{filename}", "w") as f:
                f.write(resp.text)

    if laws:
        print(f"{year}: downloaded {len(laws)} laws")
        time.sleep(0.5)  # be polite to GitHub
