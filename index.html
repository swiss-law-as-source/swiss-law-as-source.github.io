Swiss Law as Source

Swiss federal & cantonal law, version-controlled.

Every law text in DE / FR / IT, one git commit per consolidation, served as Markdown and a small static JSON API. Data is sourced from Fedlex (federal) and LexWork / LexFind (cantonal).

Repository layout

ch/{sr_number}/{de|fr|it}/{sr_number}.md   Federal laws
kt/{canton}/{de|fr|it}/{number}.md         Cantonal laws (zh, bs, ag, …)

Static API

Every endpoint is a flat JSON file on GitHub Pages. No server, no rate limit.

Publications by year — every law revision committed in that year:

GET /api/v1/publications/YYYY.json GET /api/v1/publications/today.json GET /api/v1/publications/index.json  # list of years

Law text — served by GitHub's raw endpoint:

GET https://raw.githubusercontent.com/swiss-law-as-source/swiss-law-as-source.github.io/main/{path}

Historical text — any commit hash from the publications index:

GET https://raw.githubusercontent.com/swiss-law-as-source/swiss-law-as-source.github.io/{commit}/{path}

Pre-1970 dates

Git can't represent commit timestamps before the Unix epoch, so very old laws (e.g. an 1852 treaty) live as a commit at the earliest Fedlex consolidation, but carry an original_publication_date field in their YAML frontmatter. The API surfaces them under their real publication year — so /api/v1/publications/1852.json works.
