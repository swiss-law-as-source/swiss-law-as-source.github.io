# swiss-law-as-source.github.io

Swiss federal and cantonal law as version-controlled Markdown — DE / FR / IT,
one git commit per consolidation, served as a static JSON API on GitHub Pages.

Live: <https://swiss-law-as-source.github.io>

## Layout

```
ch/{sr_number}/{de|fr|it}/{sr_number}.md   Federal laws (Fedlex SR numbers)
kt/{canton}/{de|fr|it}/{number}.md         Cantonal laws (zh, bs, ag, …)
api/v1/publications/{year}.json            All revisions for a calendar year
api/v1/publications/index.json             List of available years
api/v1/publications/today.json             Today's revisions
```

Each Markdown file has YAML frontmatter:

```yaml
---
sr_number: "210"
title: Schweizerisches Zivilgesetzbuch vom 10. Dezember 1907
language: de
version_date: "2026-07-01"
source: https://fedlex.data.admin.ch
# Only set when the law's original publication predates its earliest
# Fedlex consolidation — surfaces pre-1970 dates via the API.
# original_publication_date: '1907-12-10'
---
```

## Commit semantics

- One commit per Fedlex consolidation (its `dateApplicability`).
- One extra commit per original publication when the law's `dateDocument`
  predates the earliest consolidation and is on or after 1970-01-01.
- For pre-1970 publications, the date lives in the file's frontmatter
  (`original_publication_date`); git's `time_t` cannot represent earlier
  timestamps. The API surfaces those entries alongside the git-backed ones.
- Commit message format: `SR <number>: <title> (<version_date>)`.

## API

Static JSON, regenerated by GitHub Actions after every pipeline run. No
server, no rate limit, no authentication — fetch from any client.

```
GET /api/v1/publications/<YYYY>.json
GET /api/v1/publications/today.json
GET /api/v1/publications/index.json
```

Response shape:

```json
{
  "date_prefix": "2023",
  "count": 712,
  "publications": [
    {
      "commit_hash": "ada66b…",
      "date": "2023-01-01",
      "sr_number": "210",
      "title": "Schweizerisches Zivilgesetzbuch …",
      "scope": "federal",
      "languages": ["de", "fr", "it"],
      "files": [
        {
          "language": "de",
          "path": "ch/210/de/210.md",
          "url_main":       "https://raw.githubusercontent.com/swiss-law-as-source/swiss-law-as-source.github.io/main/ch/210/de/210.md",
          "url_at_version": "https://raw.githubusercontent.com/swiss-law-as-source/swiss-law-as-source.github.io/ada66b…/ch/210/de/210.md"
        }
      ]
    }
  ]
}
```

`url_main` always points at the latest text on `main`; `url_at_version`
is pinned to the commit that produced this revision, so the content is
byte-for-byte reproducible.

### Live (dynamic) API

If you'd rather query by arbitrary date range / filters than fetch a
per-year file, run the FastAPI server locally:

```bash
pip install -e .
legalize-ch serve   # http://localhost:8000
```

Same endpoints plus `?date=YYYY[-MM[-DD]]`, `?lang=`, `?scope=`, full-text
search, and as-of-date law retrieval.

## Updating the data

GitHub Actions runs `legalize-ch update` daily (cron `17 4 * * *` UTC),
regenerates the JSON exports, and commits back to `main`. You can also
trigger a full bootstrap manually from the Actions tab → "Update + Publish"
→ "Run workflow" → check "Run full bootstrap".

Locally:

```bash
legalize-ch bootstrap --scope all   # First-time ingest (hours)
legalize-ch update   --scope all    # Incremental since last run
legalize-ch export                  # Regenerate api/v1/publications/*.json
```

## Data sources

- **Federal** — [Fedlex SPARQL endpoint](https://fedlex.data.admin.ch/sparqlendpoint)
  (`jolux:` ontology). ~9000 laws, 17 k+ revisions.
- **Cantonal LexWork** (19 cantons: AG, AI, AR, BE, BL, BS, FR, GL, GR,
  LU, NW, OW, SG, SH, SO, TG, UR, VS, ZG) — each canton's portal serves
  the JSON catalog at `https://{canton-host}/api/{lang}/texts_of_law/lightweight_index`
  and per-law HTML body at `…/texts_of_law/{number}/show_as_json`. The
  HTML body lives inside `selected_version.json_content.document` as a
  recursive tree of nodes.
- **Cantonal LexFind** (7 cantons: GE, JU, NE, SZ, TI, VD, ZH) — these
  cantons don't run their own LexWork portal. LexFind serves their
  catalog at `/api/fe/{lang}/entities/{entity_id}/recent-changes` and
  per-law metadata at `…/texts-of-law/{tol_id}/with-version-groups`.
  The body itself is currently published only as a PDF, so each
  LexFind-only revision lands as a metadata-rich markdown stub pointing
  at the upstream PDF + portal page.

The Phase-1 portal discovery script lives at
`scripts/discover_canton_apis.py` (Playwright-based) and emits its
findings to `data/discovery/`.
